Свежий Ubuntu VPS → рабочий сервер, без ручного гугления одних и тех же команд каждый раз. Современные CLI-утилиты, Docker, nginx, zram/swap по проверенной схеме, красивый `fastfetch` + `starship`, и по желанию — SSH hardening, который **сам себя проверяет** перед тем как отключить тебе пароль.

- ⚡️ Меню вместо мануала — 15 пунктов, видно что уже стоит, что нет
- 🧠 Понимает, что уже настроено вручную, и не перезаписывает молча
- 🔒 SSH hardening с самопроверкой на живую — не отключит пароль, пока не убедится, что ключ реально работает
- 📦 Пачками по разделам (база / сервисы / защита) или всё разом — не обязательно тыкать по одному
- 🔄 Сам следит за обновлениями и обновляется по команде

> Скрипт делался для себя, чтобы не гуглить одно и то же на каждом новом сервере — но пожелания и баг-репорты только приветствуются, пишите в issues.

## Быстрый старт

1. Запустить установку:

```bash
curl -fsSL https://raw.githubusercontent.com/SkyDeaD/UbuntuServer-Fast-Configuration/main/install.sh | sudo bash
```

2. Открывается меню — выбираешь что нужно по номеру, разделом (`B`/`S`/`P`) или всё сразу (`A`).
3. Готово — дальше сервер настроен, можно продолжать работать.

Открыть меню в любой момент:

```bash
usfc
```

(алиас на `sudo usfc` ставится сам в `.bashrc` пунктом 8 — писать `sudo` вручную не нужно)

> **Перед боевым сервером** — прогони на тестовой/снэпшотящейся VPS, особенно если планируешь соглашаться на SSH hardening и UFW.

## Как работает

- **Идемпотентно.** Каждый пункт сам проверяет, применён ли он уже — и если применён иначе, чем в гайде (руками до этого настраивал), покажет разницу и спросит, а не перезапишет молча.
- **SSH hardening с самопроверкой.** Перед тем как отключить пароль, скрипт генерирует одноразовый тестовый ключ и реально логинится им через localhost — до и после применения конфига. Не сработало — автооткат, пароль остаётся включён, текущая сессия не рвётся ни на каком этапе.
- **UFW без сюрпризов.** Сначала показывает, что реально слушает порты на сервере, и только потом спрашивает — если у тебя уже висит VPN на нестандартном порту, увидишь его в списке до, а не после включения firewall.
- **`.bashrc`-алиасы не заслоняют системные команды при sudo.** `eval`-строки для zoxide/starship условные — не важно, в каком порядке ставил пункты, ошибок "command not found" не будет.

### Как добавить лого/скриншоты

Файлы в репозиторий класть не обязательно — GitHub сам хостит картинки, вставленные через веб-интерфейс:

1. Открой `README.md` на github.com → значок карандаша (Edit).
2. Перетащи картинку (или `Ctrl+V`) прямо в поле редактирования текста.
3. GitHub сам загрузит её и подставит `![...](https://github.com/user-attachments/assets/...)` на место курсора.
4. Commit changes.

Ссылка постоянная, работает и вне репозитория. Для логотипа/скриншотов, которые не хочется гонять через коммит README — тот же трюк работает в любом Issue (даже не обязательно его сохранять, ссылка на картинку остаётся рабочей).

## Кастомизация

Свой конфиг `fastfetch` — просто отредактируй `config.jsonc` в репозитории, `setup.sh` подтягивает его напрямую при каждом запуске.

Свой форк — поменяй `REPO_RAW_BASE` в начале `install.sh` и `setup.sh` на свой репозиторий/ветку.

**Важно:** при правке `setup.sh` поднимай номер в файле `VERSION` — по нему `usfc` понимает, что вышло обновление.

## Сознательно не автоматизировано

- Смена SSH-порта с 22 на нестандартный
- Блокировка пароля root — при отключённом `PermitRootLogin` избыточно
- Более тонкие правила UFW (rate-limiting, конкретные IP)
- Настройка самого VPN/прокси-стека — отдельная задача под конкретный сервер

## FAQ

**После установки конфигов новые алиасы не появились сразу.**
Скрипт работает в дочернем процессе — он физически не может изменить окружение твоей текущей SSH-сессии (это фундаментальное ограничение Unix, а не недоработка). В конце меню сам предложит открыть свежий шелл с уже применёнными алиасами — соглашайся, и делать ничего вручную не придётся. Либо просто `source ~/.bashrc` или переподключись по SSH.

**Docker не ставится, ошибка про кодовое имя дистрибутива.**
Официальный репозиторий Docker иногда отстаёт от свежих релизов Ubuntu — скрипт сам это ловит и переключается на совместимый `noble`.

**Хочу пересмотреть решение по SSH hardening/UFW без пересоздания сервера.**
`usfc` ещё раз — на уже настроенных пунктах меню покажет текущее состояние и спросит, менять ли.

## Требования

Ubuntu 24.04 или 26.04, root-доступ, исходящий интернет (apt, PPA, GitHub raw, download.docker.com, starship.rs).
